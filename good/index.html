Disclosures and Dialogs

This section demonstrates accessible disclosure widgets and dialogs patterns that work seamlessly with screen readers and keyboard navigation.

Disclosure

Disclosure widgets can reveal inline content directly below the trigger button, or floating content that appears overlaid on the page. Both patterns maintain proper focus management and screen reader announcements.

Inline Disclosure

This is inline disclosure content that appears directly below the button. It maintains the document flow and pushes other content down when expanded.

  • Preserves document flow
  • Content appears in logical reading order
  • Good for supplementary information

Floating Disclosure

This is floating disclosure content that appears as an overlay. It's positioned relative to the trigger button and doesn't affect the document flow.

  • Appears as overlay content
  • Doesn't disrupt document flow
  • Ideal for contextual help or details
  • Can be closed with Escape key or by clicking outside

Dialogs

Dialog windows are overlay elements that appear on top of the main content and require user interaction before they can return to the main interface. They are commonly used for confirmations, forms, alerts, or displaying detailed information without navigating away from the current page. Dialogs should trap focus, be dismissible, and provide clear navigation back to the triggering element.

Modal Dialogs

Modal dialogs are overlay windows that appear on top of the main content and require user interaction before they can return to the main interface. They are commonly used for confirmations, forms, alerts, or displaying detailed information without navigating away from the current page. Modals should trap focus, be dismissible, and provide clear navigation back to the triggering element.

Plain Dialogs

Plain dialogs are overlay windows that appear on top of the main content but differ from modal dialogs in that they don't necessarily prevent interaction with other parts of the page. Unlike modals (which use aria-modal="true"), plain dialogs allow screen readers to access the background content while still providing a focused interaction area. They're ideal for supplementary information, help text, or non-critical user interactions.

Accessible Modal Dialog

This is an accessible modal dialog that demonstrates proper WCAG 2.2 compliance. It includes:

  • Focus trapping within the modal
  • Proper ARIA attributes for screen readers
  • Keyboard navigation support
  • Background scroll prevention
  • Focus restoration to trigger button when closed

You can close this modal by clicking the close button, pressing Escape, or clicking outside the modal content.

Accessible Plain Dialog

This is an accessible plain dialog that demonstrates the difference from modal dialogs. Key features:

  • Does not use aria-modal="true" allowing background access
  • Proper ARIA attributes for screen readers
  • Keyboard navigation support with Escape key
  • Focus restoration to trigger button when closed
  • Can be dismissed with Escape or close button

Unlike modal dialogs, plain dialogs don't trap focus as strictly and allow screen readers to access background content when needed.
